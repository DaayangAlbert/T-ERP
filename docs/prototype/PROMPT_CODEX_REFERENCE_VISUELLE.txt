2. Ajouter PayrollEmployeeSnapshot.

Créer :

model PayrollEmployeeSnapshot {
  id              String   @id @default(cuid())
  tenantId        String
  payrollCycleId  String
  employeeKey     String
  userId          String?
  matricule       String
  firstName       String
  lastName        String
  fullName        String
  position        String?
  category        String?
  echelon         String?
  department      String?
  site            String?
  contractType    String?
  hireDate        DateTime?
  cnpsNumber      String?
  taxNumber       String?
  bankName        String?
  bankAccount     String?
  profilePhotoUrl String?
  baseSalary      Decimal  @default(0)
  createdAt       DateTime @default(now())

  payrollCycle    PayrollCycle @relation(fields: [payrollCycleId], references: [id])

  @@unique([payrollCycleId, employeeKey])

Rôle :
Cette table doit figer les informations du salarié au moment du cycle de paie.
Même si le salarié change de poste, de banque, de photo ou quitte l’entreprise, le bulletin historique doit garder les anciennes informations.

WORKFLOW
========
Statuts attendus :

DRAFT
→ CALCULATING
→ CALCULATED
→ N2_PENDING après validation RH N1
→ N3_PENDING après validation DAF N2
→ PAID ou APPROVED selon logique existante
→ DIPE_SUBMITTED
→ CLOSED

Corriger validate-n1 :
après validation RH, le cycle ne doit pas rester N1_PENDING.
Il doit passer à N2_PENDING.
